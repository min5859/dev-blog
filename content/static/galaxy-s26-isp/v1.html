5 최신 진화 방향: SFE vs ICPU

두 회사가 최신 칩에서 “추가한 블록”의 의미가 서로 다릅니다.

QUALCOMM · SFE

데이터 경로를 분화

Sensor Front End — 센서 전단 픽셀 처리(sHDR·Fast-shutter·베이어 전처리)를 IFE에서 떼어냄.

센서 → CSID
SFE
IFE
SFE→IFE
DDR

방향성: 이미지 데이터 파이프라인을 더 잘게 쪼개 멀티프레임 HDR·고속 센서를 효율 처리. 제어는 여전히 커널+CamX.

EXYNOS · ICPU

제어 지능을 오프로드

Image Control Processing Unit — 3A 연산을 전용 코프로세서 펌웨어(RTA)로 완전히 이관.

커널 (shot 전달만)
mailbox 비동기
⚙ ICPU 코프로세서
RTA — AE/AWB/AF 연산
결과 → HW 적용

방향성: 제어/알고리즘 지능을 전용 칩으로 내려 AP·커널 부하↓, 실시간 결정성↑.

방향성 요약
Qualcomm은 “데이터 경로(SFE)를 분화”, Exynos는 “제어 지능(ICPU/RTA)을 펌웨어로 오프로드”. 같은 ISP 진화지만 Qualcomm은 픽셀 흐름 최적화, Exynos는 연산 위치 재배치에 무게를 둡니다.
